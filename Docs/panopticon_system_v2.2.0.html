    1. 연출 시스템 개요

    스토리·튜토리얼 진행에 필요한 연출 기능 모음. 각 기능을 독립 컴포넌트로 구현하고, 시나리오 기획서의 행동 컬럼 값으로 호출한다.

    🚫 연출 구현 제한 — 포스트 프로세싱 기반 구현 전면 금지 Meta Quest VR 하드웨어의 GPU 한계로 인해 모든 연출에서 포스트 프로세싱(Post Processing) 기반 구현을 사용해서는 안 된다. 이는 심각한 성능 저하 및 멀미를 유발한다. 잠망경 마스킹, 화면 글리치 등 시각 효과 전부에 해당하며, 반드시 대체 구현 방식을 사용한다.

    1-1. 게임오버 연출

    💀 게임오버 연출 — 오버워치 매치종료 모티브

    개요

    오버워치의 매치 종료 연출을 모티브로 한다. 월드 틱 타임(World Tick Time)을 낮춰 시간이 느려지는 듯한 슬로모션 연출 이후 게임을 종료한다. 포스트 프로세싱 없이 UE 월드 타임 딜레이션으로만 구현한다.

    진행 순서

    1. 탈출 감지 → 게임오버 조건 충족
    2. 4초간 월드 틱 타임을 기존의 0.1 수준으로 낮춤 (슬로모션 효과)
    3. 게임오버 모델링 표시 (평상시 Hidden 상태로 씬에 배치, 이 시점에 Visible로 전환)
    4. 잭 대사 WAV 1회 재생: "오 이런 정말 실망스럽군요.. 정말로"
    5. 슬로모션 종료 → 로비 이동

    구현 방식 비고

    게임오버 모델링은 씬에 평상시 숨겨두고(SetActorHiddenInGame = true) 게임오버 시 표시하는 방식을 권장. 동적 스폰보다 기기 부하가 적다.

    게임오버 모델링 정체: 표시되는 오브젝트의 구체적 형태 (잭 얼굴 대형 화면 연출, 전광판 텍스트, 별도 3D 오브젝트 등) 미정 TBD

    참고: UE5 Global Time Dilation — dev.epicgames.com/…/time-dilation

    페이드 인-아웃

    🎬 페이드 인-아웃 — Fade In/Out

    개요

    화면이 점차 어두워지거나(Fade Out), 반대로 밝아지는(Fade In) 전환 연출. 씬 이동 앞뒤로 항상 짝으로 사용한다.

    사용 상황 예시

    스토리 챕터 1·2·3 엔딩 후 로비 복귀 / 로비에서 스테이지 진입 시 / 스테이지 클리어·게임오버 후 로비 이동 시.

    진행 순서

    1. 마지막 대사 재생 완료
    2. Fade Out — 화면 점차 어두워짐 (0.5~1초)
    3. 씬 이동 (로비 ↔ 스테이지)
    4. Fade In — 화면 점차 밝아짐 (0.5~1초)

    구체적 연출 설명

    UE5 Camera Fade(Sequencer의 Fade Track 또는 PlayerCameraManager의 StartCameraFade 함수)를 사용한다. 포스트 프로세싱 없이 Camera Component에서 직접 처리한다.

    공식문서

    UE5 공식문서 — Fade Track (Sequencer)

    잠망경 (Periscope)

    🔭 잠망경 — 신규 제압장비 (TUTO 3 최초 등장)

    개요

    천장 구역을 감시할 수 있는 유일한 장비. 스파이더맨·플로팅 등 천장 관련 죄수 상태에 대응하기 위한 필수 장비.

    사용 상황 예시

    TUTO 3에서 첫 교육. 이후 스테이지에서 천장 이탈 죄수 감지 시 플레이어가 자발적으로 사용.

    기술적 제약

    VR FOV 수정 불가: SDK 정책 위반 가능성.
    포스트프로세싱 마스킹 불가: GPU 부하 과다. (연출 금지 정책 해당)

    확정 구현 방식 — UI 마스킹 + 카메라 전환

    1. 플레이어 카메라가 잠망경 콜리전 박스와 오버랩 → 마스크 UI(검은 원) 생성, opacity 0 시작
    2. 잠망경-카메라 거리 가까워질수록 opacity 상승 (점점 어두워짐)
    3. opacity 최대치 도달 시 전환 사운드 1초 재생
    4. 사운드 종료 후 천장 고정 카메라로 전환, 마스크 UI 비활성화

    구체적 연출 설명

    마스크 UI는 UMG Widget Component(World Space)로 배치하여 포스트프로세싱 없이 화면을 덮는다. 천장 카메라는 씬에 미리 배치된 고정 카메라(Camera Actor)로 전환(SetViewTargetWithBlend)한다.

    구성 요소 상태

    요소역할상태

    글리치 연출

    ⚡ 글리치 — 죄수 글리치 / 화면 글리치 / 디스토션 보이스

    개요

    오류 상태 죄수에 적용하는 텍스처 교체, 잭 모니터 화면 흔들림, 아브리엘 보이스 디스토션의 세 가지로 구성된다. 모두 포스트프로세싱 없이 구현한다.

    사용 상황 예시

    챕터트리거유형구현 방식
    챕터 1한 사이클 클리어 직후죄수 글리치 + 에녹 디스토션(강)죄수 텍스처 교체 + 카메라 색조 Material 이상 효과 + 오디오 피치 변조
    챕터 2 — 3분게임시작레버 작동 후보이스 디스토션 (중간)오디오 피치·리버브 변조
    챕터 2 — 5분게임시작레버 작동 후보이스 디스토션 (낮음)오디오 피치 변조 약하게
    챕터 2 — 로비아브리엘 첫 접촉화면 글리치 UV Offset 대체모니터 스프라이트 UV Offset 애니메이션 (포스트프로세싱 금지로 대체)
    챕터 3글리치 죄수 탈출 → 잭 오류 감지잭 화면 흔들림 + 글리치 사운드모니터 Widget 위치 흔들림 + WAV
    챕터 3팩스 공격 성공시설 전체 흔들림 + 전기 이펙트Camera Shake + Niagara 파티클

    구체적 연출 설명

    화면 글리치는 포스트프로세싱 금지로 인해 모니터 스프라이트의 Material UV Offset 애니메이션으로 대체한다. 모니터 텍스처 자체가 일그러지는 효과를 Material 파라미터 타임라인으로 구현한다.

    Meta Cloud Backup API 연동

    Meta Platform SDK의 Cloud Storage(KeyValue 저장소) 또는 Cloud Backup 기능을 통해 클라우드 데이터를 동기화한다.

    공식문서 참고
    Meta Cloud Backup — Unreal Engine

    주요 흐름:
    1. 게임 시작 시 UPlatformCloudStorage::LoadData() 호출 → 클라우드 최신 데이터 로드
    2. 클리어 이벤트 발생 시 SaveNow() → UPlatformCloudStorage::SaveData() 호출
    3. 충돌 시(로컬 vs 클라우드) 더 높은 클리어 수 기준으로 병합 처리
    4. 오프라인 상태에서는 로컬 저장 후 온라인 복귀 시 자동 동기화
    Meta Cloud Storage는 Quest 디바이스 기준으로 계정당 용량 제한이 있다. 무한모드 이력 전체를 클라우드에 저장할 경우 초과 가능성이 있어 로컬 전용으로 분리함.

    3-3. 저장 시점 판단 기준

    상황저장 여부판단 근거
    게임 최초 시작해당 없음저장 데이터 없음
    튜토리얼 진행 중 종료저장 안 함튜토리얼은 선택 사항 → 재시작해도 불쾌하지 않음
    튜토리얼 클리어즉시 저장TutorialClearedFlags 업데이트 + 테이프 사출
    스테이지 클리어즉시 저장StageClearedFlags 업데이트 + 테이프 사출
    스테이지 진행(연출) 중 종료재시도 시 연출 재재생자의·비자의 종료 모두 동일하게 처리
    무한모드 종료즉시 저장점수·이력·최고기록 갱신
    설정 변경즉시 저장로컬 GameSettings 업데이트

    3-4. 모드별 종료 시 저장 작동 예시

    ① 튜토리얼 중간 종료

    TUTO 2 진행 중 헤드셋 벗음 → 앱 종료
    → 저장 없음. 다음 실행 시 로비에서 T_TUT_02 테이프 재선택 후 처음부터 재시작.

    ② 튜토리얼 클리어

    TUTO 2 클리어 → TutorialClearedFlags[2] = true → OwnedTapeIDs에 T_TUT_03 추가
    → SaveNow() (클라우드) → T_TUT_03 테이프 사출 연출.

    ③ 스테이지 연출 중 종료

    챕터 2 클리어 후 아브리엘 첫 접촉 대사 재생 중 종료
    → StageClearedFlags[2]는 이미 저장됨. 다음 실행 시 로비에서 T_STG_03 보유 상태.
    → 아브리엘 대사는 다시 보지 않아도 됨 (이미 클리어된 테이프로 재생 시 재시청 가능).

    ④ 무한모드 종료 (도중 종료)

    무한모드 10분 진행 중 죄수 탈출 → 게임오버 → 로비 이동
    → RunHistory에 기록 추가 (Score, SurvivalSeconds, EndReason)
    → BestScore·BestSurvivalSeconds 갱신 여부 확인
    → SaveNow() (클라우드 + 로컬).

    ⑤ 무한모드 도중 강제 종료

    무한모드 진행 중 갑자기 앱 종료 (배터리 방전 등)
    → RunHistory 기록 없음. 해당 플레이 기록 유실.
    → 다음 실행 시 이전 저장 상태 그대로 복원.

    테이프 사출 순서표

    클리어 이벤트Flag 업데이트사출 테이프
    Tutorial i 완료 (i=1~3)TutorialClearedFlags[i] = trueT_TUT_(i+1)
    Tutorial 4 완료TutorialClearedFlags[4] = trueT_STG_01
    Stage i 완료 (i=1~3)StageClearedFlags[i] = trueT_STG_(i+1)
    Stage 4 완료StageClearedFlags[4] = trueT_INF_01 + bInfiniteUnlocked = true
    중복 사출 방지: PlayingTapeID가 이미 OwnedTapeIDs에 존재하면 클리어 이벤트 미발동.
    T_TUT_01 초기 제공: 게임 최초 시작 시 OwnedTapeIDs에 T_TUT_01이 기본값으로 포함된 상태로 초기화된다. 별도 사출 연출 없이 처음부터 로비 테이프 슬롯에 배치되어 있는 것으로 처리한다.
